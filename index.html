Advanced Spawner

Spawn frogs faster as the score increases
🧠 Logic How the spawner works ▾

The Spawner script uses Unity's built-in Coroutines — a way to pause and resume code over time. Every few seconds, it asks the GameManager for the current score, then calculates a shorter and shorter wait time as the score rises. Think of it like a treadmill that gradually speeds up.

The formula is: spawnInterval = baseInterval / (1 + score * difficultyMultiplier)

Beginner Tip: A Coroutine (IEnumerator) is a special function in Unity that can pause itself using yield return new WaitForSeconds(x). It's perfect for "do something every N seconds" logic.
🖱️ Editor Steps Setting up the Spawner in Unity ▾
  1. In the Hierarchy window, right-click and select Create Empty. Rename it to Spawner.
  2. In the Inspector, click Add Component and type Spawner, then select New Script.
  3. Open the script in your code editor and replace the contents with the code below.
  4. Create empty GameObjects named SpawnPointLeft and SpawnPointRight off-screen at the edges of the scene.
💻 Code Spawner.cs — Complete script ▾
Spawner.cs 
using System.Collections;
using UnityEngine;

// This script controls when and how fast frogs are spawned.
// Attach it to an empty GameObject called "Spawner" in your scene.
public class Spawner : MonoBehaviour
{
    // ── PUBLIC VARIABLES ──────────────────────────────────────────
    // These appear in the Inspector so you can tweak them without
    // touching the code. Great for balancing your game!

    [Header("Frog Prefabs")]
    public GameObject normalFrogPrefab;   // drag your Normal Frog prefab here
    public GameObject fastFrogPrefab;     // drag your Fast Frog prefab here
    public GameObject tankFrogPrefab;     // drag your Tank Frog prefab here

    [Header("Spawn Points")]
    public Transform spawnPointLeft;      // left edge of screen (off-screen)
    public Transform spawnPointRight;     // right edge of screen (off-screen)

    [Header("Difficulty Settings")]
    public float baseInterval = 2.5f;   // seconds between spawns at score=0
    public float minInterval = 0.4f;    // never spawn faster than this
    public float difficultyMultiplier = 0.02f; // how fast it speeds up

    // ── PRIVATE VARIABLES ────────────────────────────────────────
    private Coroutine spawnCoroutine;    // reference to our running coroutine

    // ── START() ───────────────────────────────────────────────────
    // Unity calls Start() exactly ONCE when the game begins.
    // We use it to kick off our repeating spawn loop.
    void Start()
    {
        spawnCoroutine = StartCoroutine(SpawnLoop());
    }

    // ── SPAWN LOOP ────────────────────────────────────────────────
    // This Coroutine runs forever (while the game is playing).
    // It waits, spawns a frog, then waits again with a shorter interval.
    IEnumerator SpawnLoop()
    {
        while (true)
        {
            // 1. Get the current score from the GameManager
            int score = GameManager.instance.score;

            // 2. Calculate how long to wait before the next frog
            // Higher score = smaller interval = frogs spawn faster
            float interval = baseInterval / (1f + score * difficultyMultiplier);
            interval = Mathf.Max(interval, minInterval); // clamp to minimum

            // 3. Wait for that amount of time, then continue
            yield return new WaitForSeconds(interval);

            // 4. Only spawn if the game is still running
            if (GameManager.instance.isGameOver) yield break;

            // 5. Spawn a random frog type
            SpawnRandomFrog();
        }
    }

    // ── SPAWN RANDOM FROG ─────────────────────────────────────────
    // Picks a random frog prefab and a random spawn side, then
    // creates (Instantiates) a copy of that prefab in the scene.
    void SpawnRandomFrog()
    {
        // Pick a random number: 0=Normal, 1=Fast, 2=Tank
        // Adjust these weights to control how often each type appears
        int roll = Random.Range(0, 10);
        GameObject chosenPrefab;

        if (roll < 5)      chosenPrefab = normalFrogPrefab; // 50% chance
        else if (roll < 8) chosenPrefab = fastFrogPrefab;  // 30% chance
        else               chosenPrefab = tankFrogPrefab;  // 20% chance

        // Pick a random side to spawn from
        Transform spawnPoint = (Random.value > 0.5f)
            ? spawnPointLeft
            : spawnPointRight;

        // Instantiate = "create a copy of this prefab in the scene"
        Instantiate(chosenPrefab, spawnPoint.position, Quaternion.identity);
    }
}

🧠 Logic Prefabs and frog data ▾

A Prefab is a reusable template for a GameObject. You design one frog, save it as a Prefab, and the Spawner can create many copies. Each frog type gets the same FrogController script but with different public variable values set in the Inspector — speed, health, score value.

Key concept: One script, many behaviors. The same FrogController.cs runs all three frog types. Only the data (speed, health) is different — set per-prefab in the Inspector.
🖱️ Editor Steps Creating the three frog Prefabs ▾
  1. In the Hierarchy, right-click 2D Object › Sprite to create a sprite. Rename it NormalFrog. Assign your frog sprite in the Inspector's Sprite Renderer.
  2. With NormalFrog selected, click Add Component in the Inspector and add a Rigidbody 2D. Set Gravity Scale to 0.
  3. Add another component: Add Component › Box Collider 2D and click Edit Collider to fit it to the frog sprite.
  4. Add the FrogController script (create it via Add Component → New Script).
  5. Drag the NormalFrog from the Hierarchy into the Project window (Assets folder). Unity will ask "Create Original Prefab" — click that. Delete the object from the Hierarchy.
  6. Duplicate the Prefab twice in the Project window: rename to FastFrog and TankFrog. Double-click each to open Prefab Edit Mode and adjust the Inspector values (speed, health, color).
💻 Code FrogController.cs — Complete script ▾
FrogController.cs 
using UnityEngine;

// Attach this script to every frog Prefab.
// Change the public variables per-prefab in the Inspector!
public class FrogController : MonoBehaviour
{
    // ── PUBLIC VARIABLES ──────────────────────────────────────────
    [Header("Frog Stats")]
    public float moveSpeed = 2f;        // Normal=2, Fast=5, Tank=1.5
    public int   health    = 1;          // Normal=1, Fast=1, Tank=2
    public int   scoreValue = 10;       // points awarded when killed
    public GameObject splashEffectPrefab; // particle effect on death (optional)
    public AudioClip  ribbitSound;        // sound on click (optional)

    [Header("Screen Bounds")]
    public float destroyAtX = 15f;      // how far off-screen before "missed"

    // ── PRIVATE VARIABLES ─────────────────────────────────────────
    private Vector2     moveDirection;   // left or right?
    private Rigidbody2D rb;
    private AudioSource audioSource;
    private int         currentHealth;   // tracks remaining HP

    // ── START() ───────────────────────────────────────────────────
    // Called ONCE when the frog is first created (Instantiated).
    // We grab components and decide which direction to move.
    void Start()
    {
        rb = GetComponent<Rigidbody2D>();
        audioSource = GetComponent<AudioSource>();
        currentHealth = health;

        // If frog spawned on the left, move right; if on the right, move left.
        // We check its X position relative to the screen center (X=0).
        if (transform.position.x < 0)
            moveDirection = Vector2.right;
        else
            moveDirection = Vector2.left;

        // Flip the sprite to face the correct direction
        if (moveDirection == Vector2.left)
            transform.localScale = new Vector3(-1, 1, 1);
    }

    // ── UPDATE() ──────────────────────────────────────────────────
    // Called EVERY FRAME (60+ times per second). We use it to:
    // 1. Move the frog across the screen
    // 2. Check if it has left the screen (player missed it)
    void Update()
    {
        // Move the frog smoothly. Time.deltaTime ensures movement is
        // frame-rate independent (same speed on any computer).
        rb.linearVelocity = moveDirection * moveSpeed;

        // Check if the frog has crossed the screen edge
        if (Mathf.Abs(transform.position.x) > destroyAtX)
        {
            // Tell the GameManager the player missed a frog → lose a life
            GameManager.instance.LoseLife();

            // Remove this frog from the scene
            Destroy(gameObject);
        }
    }

    // ── ON MOUSE DOWN ─────────────────────────────────────────────
    // Unity calls this when the player clicks on this object's Collider.
    // This is the simplest way to detect clicks in 2D!
    void OnMouseDown()
    {
        TakeDamage(1);
    }

    // ── TAKE DAMAGE ───────────────────────────────────────────────
    // Reduces health. If health hits 0, the frog dies.
    // The Tank frog has health=2, so it needs 2 clicks.
    public void TakeDamage(int amount)
    {
        currentHealth -= amount;

        // Play ribbit sound on each click
        if (ribbitSound != null && audioSource != null)
            audioSource.PlayOneShot(ribbitSound);

        // Visual feedback: briefly flash red (we scale the sprite)
        StartCoroutine(FlashRed());

        if (currentHealth <= 0)
            Die();
    }

    // ── DIE ───────────────────────────────────────────────────────
    void Die()
    {
        // Add score to the GameManager
        GameManager.instance.AddScore(scoreValue);

        // Spawn the splash particle effect at the frog's position
        if (splashEffectPrefab != null)
            Instantiate(splashEffectPrefab, transform.position, Quaternion.identity);

        // Remove the frog from the scene
        Destroy(gameObject);
    }

    // ── FLASH RED ─────────────────────────────────────────────────
    // A simple hit-flash effect using the SpriteRenderer color.
    System.Collections.IEnumerator FlashRed()
    {
        SpriteRenderer sr = GetComponent<SpriteRenderer>();
        sr.color = Color.red;
        yield return new WaitForSeconds(0.1f);
        sr.color = Color.white;
    }
}

Game Manager & Scoring

Singleton pattern for global score and lives tracking
🧠 Logic What is a Singleton? ▾

A Singleton is a pattern where only ONE instance of a script can ever exist, and every other script can access it globally using GameManager.instance — like a global scoreboard on the wall that anyone can read or update.

The GameManager tracks:

  • score — increases when a frog is clicked
  • lives — decreases when a frog escapes
  • isGameOver — a flag checked by other scripts
Why use a Singleton? Without it, every script that needed the score would have to "find" the GameManager in the scene, which is slow and messy. The Singleton gives instant access from anywhere.
🖱️ Editor Steps Creating the GameManager object ▾
  1. In the Hierarchy, right-click and select Create Empty. Rename it GameManager.
  2. In the Inspector, click Add Component and create a new script called GameManager. Paste the code below.
  3. This object must be in every scene. Or better: in the first scene, check Don't Destroy On Load (already handled in the script).
💻 Code GameManager.cs — Complete script ▾
GameManager.cs 
using UnityEngine;
using UnityEngine.SceneManagement; // needed for RestartGame()

// The GameManager is the "brain" of your game.
// It tracks score, lives, and game state.
// Uses the Singleton pattern so any script can access it.
public class GameManager : MonoBehaviour
{
    // ── SINGLETON INSTANCE ────────────────────────────────────────
    // 'instance' is a static variable — it belongs to the CLASS,
    // not to any specific object. That's what lets us do
    // GameManager.instance from anywhere in our code.
    public static GameManager instance;

    // ── PUBLIC VARIABLES ──────────────────────────────────────────
    [Header("Game Settings")]
    public int startingLives = 3;

    // ── STATE VARIABLES ───────────────────────────────────────────
    // HideInInspector keeps these out of the Inspector (they're runtime data)
    [HideInInspector] public int  score;
    [HideInInspector] public int  lives;
    [HideInInspector] public bool isGameOver;

    // ── UI REFERENCES ─────────────────────────────────────────────
    [Header("UI References")]
    public UIManager uiManager; // we'll create this in Section 4

    // ── AWAKE() ───────────────────────────────────────────────────
    // Awake() runs BEFORE Start() — perfect for setting up the Singleton.
    // It runs even if the script is disabled.
    void Awake()
    {
        // SINGLETON PATTERN:
        // If no instance exists yet, we become the instance.
        if (instance == null)
        {
            instance = this;
            // Persist this object across scene loads
            DontDestroyOnLoad(gameObject);
        }
        else
        {
            // A GameManager already exists! Destroy this duplicate.
            Destroy(gameObject);
        }
    }

    // ── START() ───────────────────────────────────────────────────
    // Initialize game state when the scene loads.
    void Start()
    {
        score      = 0;
        lives      = startingLives;
        isGameOver = false;

        // Tell the UI to display the starting values
        uiManager?.UpdateScore(score);
        uiManager?.UpdateLives(lives);
    }

    // ── ADD SCORE ─────────────────────────────────────────────────
    // Called by FrogController.Die() when a frog is clicked to death.
    public void AddScore(int amount)
    {
        if (isGameOver) return; // ignore if game already over

        score += amount;

        // Update the score display on the HUD
        uiManager?.UpdateScore(score);

        Debug.Log("Score: " + score);
    }

    // ── LOSE LIFE ─────────────────────────────────────────────────
    // Called by FrogController when a frog escapes the screen.
    public void LoseLife()
    {
        if (isGameOver) return;

        lives--;
        uiManager?.UpdateLives(lives);

        Debug.Log("Lives remaining: " + lives);

        if (lives <= 0)
            TriggerGameOver();
    }

    // ── TRIGGER GAME OVER ─────────────────────────────────────────
    void TriggerGameOver()
    {
        isGameOver = true;
        Time.timeScale = 0f; // pause the game (timeScale=0 stops Update() calls)
        uiManager?.ShowGameOverScreen(score);
        Debug.Log("GAME OVER! Final score: " + score);
    }

    // ── RESTART GAME ──────────────────────────────────────────────
    // Called by the Restart button on the Game Over screen.
    public void RestartGame()
    {
        Time.timeScale = 1f; // unpause before reloading
        // Reload the current scene (index 0 = your main game scene)
        SceneManager.LoadScene(SceneManager.GetActiveScene().buildIndex);
    }
}

User Interface (UI)

HUD with Score/Lives and a Game Over screen
🧠 Logic How Unity's UI system works ▾

Unity's UI elements live on a Canvas — a special "overlay" that renders on top of everything else. Inside the Canvas we add Text components (for score/lives) and a Panel for the Game Over screen.

The UIManager script holds references to these UI elements and provides simple methods that the GameManager calls to update them.

Best practice: Keep UI code separate from game logic. The GameManager doesn't know how to draw text — it just calls uiManager.UpdateScore(score) and the UIManager handles the display.
🖱️ Editor Steps Building the HUD in the Unity Editor ▾

Creating the Canvas & HUD

  1. In the Hierarchy, right-click UI › Canvas. Unity auto-creates a Canvas and an EventSystem. In the Canvas Inspector, set Render Mode to Screen Space - Overlay.
  2. Right-click the Canvas in the Hierarchy UI › Legacy › Text. Rename it ScoreText. In the Inspector, set its Anchor to top-left, position to (10, -10), and default text to Score: 0.
  3. Duplicate ScoreText (Ctrl+D), rename it LivesText. Move its anchor to top-right, default text to Lives: 3.

Creating the Game Over Panel

  1. Right-click the Canvas UI › Panel. Rename it GameOverPanel. Set its color to black with ~80% alpha for a darkening overlay.
  2. Inside GameOverPanel, add a Text via UI › Legacy › Text. Name it GameOverText. Set text to GAME OVER, font size 60, centered.
  3. Add another Text inside the panel: FinalScoreText. Text: Score: 0, size 30.
  4. Add a Button: right-click GameOverPanel UI › Legacy › Button. Name it RestartButton. Change its child Text to Play Again.
  5. Disable the GameOverPanel GameObject (uncheck the checkbox at the top of its Inspector). We only want to show it on game over.

Adding the UIManager Script

  1. Create an empty GameObject in the Hierarchy, name it UIManager.
  2. Add a new script component called UIManager and paste the code below.
💻 Code UIManager.cs — Complete script ▾
UIManager.cs 
using UnityEngine;
using UnityEngine.UI; // required for Text, Button, etc.

// UIManager handles all visual UI updates.
// The GameManager calls these methods — UIManager just displays the data.
public class UIManager : MonoBehaviour
{
    // ── HUD REFERENCES ────────────────────────────────────────────
    [Header("HUD Elements")]
    public Text scoreText; // drag ScoreText here
    public Text livesText; // drag LivesText here

    // ── GAME OVER SCREEN REFERENCES ───────────────────────────────
    [Header("Game Over Screen")]
    public GameObject gameOverPanel;    // drag GameOverPanel here
    public Text       finalScoreText;  // drag FinalScoreText here
    public Button     restartButton;   // drag RestartButton here

    // ── START() ───────────────────────────────────────────────────
    void Start()
    {
        // Make sure the Game Over panel is hidden at the start
        gameOverPanel.SetActive(false);

        // Wire up the Restart button click event via code
        // This calls GameManager.RestartGame() when the button is pressed
        restartButton.onClick.AddListener(GameManager.instance.RestartGame);
    }

    // ── UPDATE SCORE ──────────────────────────────────────────────
    // Called by GameManager.AddScore()
    public void UpdateScore(int newScore)
    {
        scoreText.text = "Score: " + newScore;
    }

    // ── UPDATE LIVES ──────────────────────────────────────────────
    // Called by GameManager.LoseLife()
    public void UpdateLives(int remainingLives)
    {
        // Show heart emojis for lives — fun and readable!
        string hearts = "";
        for (int i = 0; i < remainingLives; i++)
            hearts += "❤ ";

        livesText.text = "Lives: " + hearts;
    }

    // ── SHOW GAME OVER SCREEN ─────────────────────────────────────
    // Called by GameManager.TriggerGameOver()
    public void ShowGameOverScreen(int finalScore)
    {
        // Activate the overlay panel
        gameOverPanel.SetActive(true);

        // Display the final score
        finalScoreText.text = "Final Score: " + finalScore;
    }
}

Juice & Feedback

Particle splash effects and ribbit audio
🧠 Logic What is "Juice" in game dev? ▾

"Juice" refers to all the small visual and audio feedback that makes a game feel good to play — screen shake, sound effects, particles, animations. A clicked frog that just disappears feels unsatisfying; one that bursts into green particles with a ribbit sound feels great.

We'll add two kinds of juice: a Particle System for a splash effect, and an Audio Source for ribbit sounds. Both are already referenced in the FrogController script from Section 2.

🖱️ Editor Steps Creating the Splash Particle Effect ▾
  1. In the Hierarchy, right-click Effects › Particle System. Rename it SplashEffect.
  2. In the Inspector's Particle System component, set these values in the main module (top section):
    • Duration: 0.5
    • Looping: OFF (unchecked)
    • Start Lifetime: 0.5
    • Start Speed: 5
    • Start Size: 0.2
    • Start Color: Green (#3FB950)
    • Gravity Modifier: 1
  3. Expand Emission: set Rate over Time to 0, and add a Burst with Count = 20 at Time 0.
  4. Expand Shape: set Shape to Sphere, Radius to 0.1.
  5. Check Stop Action at the bottom of the main module — set it to Destroy. This deletes the Particle System after it's done playing!
  6. Drag the SplashEffect from the Hierarchy into the Project window to save it as a Prefab. Then delete it from the Hierarchy.
  7. Open each Frog Prefab (double-click in Project). In FrogController's Inspector, drag the SplashEffect Prefab into the Splash Effect Prefab field.
🖱️ Editor Steps Adding Ribbit Audio ▾
  1. Find or download a ribbit sound effect (.wav or .mp3). Free sources: freesound.org or pixabay.com/sound-effects. Search "frog ribbit".
  2. Drag the audio file into your Unity Project window (Assets folder). It becomes an Audio Clip asset.
  3. Open a Frog Prefab. Click Add Component › Audio › Audio Source.
  4. In the Audio Source component: uncheck Play On Awake (we play it manually via script). Leave the AudioClip field blank.
  5. In the FrogController component on the same Prefab, drag your ribbit audio clip into the Ribbit Sound field.
  6. Repeat steps 3–5 for each frog Prefab (or use different ribbit clips per type!).
How it works: The FrogController script already calls audioSource.PlayOneShot(ribbitSound) inside TakeDamage(). PlayOneShot plays the clip once without interrupting other sounds — perfect for rapid clicking.
💻 Bonus Optional: Simple Camera Shake on Life Lost ▾

Attach this to your Main Camera for extra impact when a frog escapes.

CameraShake.cs 
using System.Collections;
using UnityEngine;

// Attach to the Main Camera. Call CameraShake.instance.Shake() from anywhere.
public class CameraShake : MonoBehaviour
{
    public static CameraShake instance;

    public float shakeDuration  = 0.3f;
    public float shakeMagnitude = 0.15f;

    private Vector3 originalPosition;

    void Awake()
    {
        instance = this;
        originalPosition = transform.localPosition;
    }

    // Call this from GameManager.LoseLife() like:
    // CameraShake.instance?.Shake();
    public void Shake()
    {
        StartCoroutine(ShakeCoroutine());
    }

    IEnumerator ShakeCoroutine()
    {
        float elapsed = 0f;

        while (elapsed < shakeDuration)
        {
            // Randomly offset the camera's position each frame
            float x = Random.Range(-1f, 1f) * shakeMagnitude;
            float y = Random.Range(-1f, 1f) * shakeMagnitude;

            transform.localPosition = originalPosition + new Vector3(x, y, 0);

            elapsed += Time.deltaTime;
            yield return null; // wait one frame, then continue
        }

        // Reset to original position when done
        transform.localPosition = originalPosition;
    }
}
To activate: Add CameraShake.instance?.Shake(); inside GameManager.LoseLife() after the lives-- line.

Unity Lifecycle Quick Reference

The order Unity calls your script methods
Awake()
First thing called when the object is created. Runs even if the script is disabled. Use for: Singleton setup, getting components.
Start()
Called once just before the first Update(). Only runs if the script is enabled. Use for: initializing variables, starting Coroutines.
Update()
Called every single frame (60+ times/sec). Use for: movement, input reading, checking conditions. Always multiply by Time.deltaTime for smooth motion.
FixedUpdate()
Called at a fixed time step (50/sec by default). Use for: physics calculations with Rigidbody. More reliable than Update() for physics.
OnMouseDown()
Called when the player clicks on this object's Collider. The easiest way to detect click/tap input on 2D objects. No extra input code needed!
OnDestroy()
Called just before this object is removed from the scene. Use for: cleanup, unsubscribing from events, saving data.
Order of execution: Awake → OnEnable → Start → Update (every frame) → FixedUpdate (fixed interval) → OnDisable → OnDestroy
